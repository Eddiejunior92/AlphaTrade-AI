# AlphaTrade AI v2

Production-ready autonomous multi-LLM high-frequency trading agent.

## Architecture
- **Backend** (`backend/`, port 3001): Node/Express + WebSocket (`/ws`).
  - `server.js` — REST API + WS broadcaster. Operator-token auth on all mutation endpoints; LIVE-mode switch refused entirely if `OPERATOR_TOKEN` is unset.
  - `strategies.js` — registry of strategies (day, swing) with their cadences, stops/targets, risk, and overnight rules.
  - `agent.js` — base 60s cycle; runs each enabled strategy on its own cadence (day=60s/1Min, swing=300s/15Min). Global trading lock prevents cycle/flatten/mode-switch from overlapping. Day strategy auto-flattens 5m before close; swing holds overnight. Daily reset at 13:30 UTC, auto-resume on restart.
  - `services/llmService.js` — 4-model ensemble: Gemini 2.0 Flash, Claude 3.7 Sonnet, GPT-4o (via OpenRouter) + Grok 4 Fast (xAI direct). Requires ≥3/4 valid responses.
  - `services/riskManager.js` — strategy-aware: takes a `strategyConfig` per evaluation. 85% confidence + 3-of-4 quorum hard gates. Refuses averaging-in (no re-buying same symbol in same strategy). Cross-strategy mark-to-market for circuit-breaker math. $100/day USD loss budget. **Compounding Confidence-Weighted sizing** (`computeDynamicScaling` + `computeTargetRisk`): per-trade $ risk = `lerp(min, max, confFraction) × growthMult × perfMult`, hard-capped at 2× scale.maxRiskUSD AND `maxPositionPct × equity` AND cash. `growthMult` steps +5% per +10% equity (range 0.5–2.0×). `perfMult` from last-20 closed-trade net PnL as % of starting balance (range 0.8–1.2×). `confFraction` lerps signal confidence within `[threshold, 1.0]`.
  - `services/alpacaService.js` — runtime paper/live switching via `setMode()`. Live mode reads separate `ALPACA_LIVE_API_KEY` / `ALPACA_LIVE_SECRET_KEY`; refuses live without them.
  - `services/ibkrService.js` — IBKR Client Portal Web API broker for ASX symbols. Mock mode when `IBKR_BASE_URL`/`IBKR_ACCOUNT_ID` unset (matches Alpaca's mock pattern). Same kill-switch hook (`setKillSwitchActive`) as Alpaca so the operator kill cascade halts ASX too.
  - `services/brokerRouter.js` — per-symbol broker routing (US→Alpaca, ASX→IBKR via `marketRegistry`). Cross-broker `closeAllPositionsAllBrokers`, `cancelAllOpenOrdersAllBrokers`, `setKillSwitchActiveAll` so operator emergency controls fan out to both. `getBars` is broker-agnostic — bars are normalized to `{t,o,h,l,c,v}` by each service.
  - `services/marketRegistry.js` — market metadata for every symbol. ASX watchlist (10 liquid blue-chips: CBA, BHP, CSL, MQG, WES, RIO, FMG, TLS, WOW, ANZ). `isAsxOpen()` and `nextAsxOpen()` use Sydney timezone math (Mon–Fri, 10:00–16:00 AEST/AEDT — no holiday calendar; ASX-holiday orders fail at the broker and are audit-tagged).
  - `services/fxService.js` — cached AUD→USD rate. Source order: `AUDUSD_RATE` env override → `exchangerate.host` (1h TTL) → hardcoded fallback `0.66`. Returns `{audusd, source, stale, fetchedAt}` for the dashboard so operators can see when FX is stale or has fallen back.
  - `services/brokerService.js` — "Alpha" personal-broker chat (Claude 3.7 via OpenRouter), strategy/mode-aware.
  - `services/db.js` — PostgreSQL with `ensureSchema()` migration. `holdings` has composite PK `(symbol, strategy)` so the same symbol can be held in both strategies independently.
  - `services/discordService.js` — webhook alerts.
  - `services/sentimentService.js` — Grok-powered news sentiment per symbol. Returns `{score: -1..+1, label, summary, insights, sources}`. Cached 30 min per symbol; refreshed in background each cycle. Injected into the LLM ensemble prompt and exposed to the dashboard.
  - `services/adaptiveLearningService.js` — nightly recompute (22:00 UTC) of per-(symbol,strategy) and per-(model,strategy) win-rate from closed trades. Exposes `getCalibrationHints()` (prompt block, advisory only) and `getSizingMultiplier()` ∈ [0.7, 1.2]. Never relaxes quorum/gate.
  - `services/portfolioOptimizationService.js` — correlation-aware portfolio risk. `evaluateAddition()` returns `sizeMult` ∈ [0.5, 1.0] based on existing-holding correlation + concentration. Pure SIZING modifier, applied AFTER quorum/gate.
  - `services/hedgingService.js` — portfolio-level risk monitor. Posts Discord alert on risk spike; if `AUTO_HEDGE=true`, places SH (inverse SPY) buy at 15% of long exposure. Cooldown-protected, never bypasses circuit breaker. Default OFF (advisory).
  - `services/orderFlowService.js` — bar-derived buy/sell pressure proxy (volume-weighted up vs down candles). Lightweight prompt block.
  - `services/optionsActivityService.js` — Grok-fetched unusual options activity flag per symbol. Cached, refreshed every 30 min during market hours. Prompt block only — never sizes trades.
  - `services/macroFactorService.js` — cross-asset macro factor reader. Pulls 12 ETF proxies (SHY/IEF/TLT for the rate curve, TIP for inflation, VXX for vol, UUP for USD, GLD/USO/DBC for commodities, HYG for credit, SPY/EEM for equity risk) via Alpaca's free daily-bar feed (no extra API keys). Computes 1d/5d/20d log-returns + 20d realized vol per factor, then derives composites: yield-curve change (TLT-SHY 5d), breakeven inflation (TIP-IEF 20d), credit-equity divergence (HYG-SPY 5d), and a risk-on/off composite (avg of equity/EM/credit minus avg of VXX/USD/gold, 5d). 60-min in-memory TTL with in-flight dedup. Read-only — failures degrade to nulls.
  - `services/macroForecastService.js` — forward-looking macro regime forecast layer. Classifies BOTH the current macro regime AND the most likely regime over the next 24-48h with a confidence score, picking from {VOL_SPIKE, RATE_SHOCK, RISK_OFF, STAGFLATION, INFLATION_RISING, USD_STRONG, RISK_ON, GOLDILOCKS, NEUTRAL}. Tiebreak prioritises safety regimes (vol/rate/risk-off > constructive). Forecast is momentum-based: 5d-and-20d aligned signals project forward with higher confidence; 5d-only signals get downgraded with a "potential mean-reversion" flag. Derives safety-preserving adjustments per regime: `confidenceBoost ∈ [0, 0.05]` (added to the strategy's confidence threshold inside `riskManager.checkQuorum`, additive with `metaLearningService`'s regimeBoost — pure tightening) and `sizeMult ∈ [0.70, 1.00]` (multiplied alongside the existing mults inside `riskManager.evaluateBuy` — **hard ceiling of 1.0, macro can NEVER amplify size**). 30-min cache + cold-start identity. Refreshed every 60 min during US market hours after a 75s post-boot warm-up. Injected as the `macroForecast` block in every LLM prompt (~3-5 lines: now/forecast labels with confidences, brief reasoning, risk-posture summary, advisory voice hint). Applies to BOTH US and ASX symbols (cross-asset signals are global). Strictly informational at the LLM layer; quorum (3-of-4), confidence gate, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, and trailing-stop ratchet are ALL untouched and retain full veto power. Endpoint: `GET /api/macro-forecast`.
  - `services/scenarioSimService.js` — self-play Monte-Carlo scenario simulation. Pure-JS GBM path generator (configurable via `SIM_PATHS`, default 400 paths × 3 trading-day horizon) that fuses recent bars + ATR% + IV (when present) + regime + macro forecast into a probability-weighted forward outlook. Vol blends three sources (realized per-bar → daily, ATR%, IV/√252) with adaptive weights (IV gets 1.2× since it's forward-looking) then multiplied by a regime-conditional vol multiplier ∈ [0.8, 1.6] (high-vol/VOL_SPIKE/RATE_SHOCK expand tails; GOLDILOCKS dampens). Drift = OLS log-slope of last 30 closes per-day, capped at ±0.4%/day, plus regime+macro tilt (asymmetric — adverse regimes pull harder than constructive ones lift). Optional jump-diffusion overlay during stress regimes (10%/day chance of ±2-3% Laplace-tail jump, biased negative in RISK_OFF/STAGFLATION). Computes per-day median/5th/95th percentiles, P(up≥1%/2%), P(down≥1%/2%), P(strategy stop hit), P(strategy target hit), and 3d expected return — all derived from the actual path bundle. 3-min cache keyed on lastBarT so re-runs in the same minute hit cache. Renders 3-5 line prompt block injected as the `scenarioSim` block in every LLM prompt (both day + swing). Strictly informational — does NOT vote, size, or gate; quorum (3-of-4), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, and trailing-stop ratchet ALL retain full veto power. Endpoint: `GET /api/scenario-sim/:symbol`.
  - `services/llmWeightingService.js` — dynamic LLM ensemble weighting layer. Learns optimal per-model weights per (strategy × regime × market) from realised P&L attributed to the originating BUY's voting models. New `model_regime_performance` table stores the finer-grained context buckets alongside the existing (model × strategy) `model_performance` table. Resolution order per model: full-trust on the (model, strategy, regime, market) bucket when n ≥ 12, else linear blend with the (model, strategy) bucket weighted by sample-count vs trust floor, else strategy-only, else uniform 1.0 (cold-start). Bayesian-smoothed win-rate (prior n=8 trades at 50% wr) → weight ∈ [0.5, 1.5], renormalised so Σweights = model count (keeps weighted vote tally on the same numeric scale as raw count). 60-second cache, busted on every recordOutcome write so the very next cycle sees fresh weights. Wired into `adaptiveLearningService.recordOutcome` to update on every closed trade with regime + market resolved from the originating TRADE_EXECUTED audit row (best-effort, never blocks). Endpoint: `GET /api/llm-weights?strategy=&regime=&market=`.
  - `services/metaReasonerService.js` — final meta-reasoner synthesis layer. After the 4 ensemble models vote, runs ONE additional Claude 3.7 Sonnet call (configurable via `META_REASONER_MODEL`) that's given the raw votes + dynamic ensemble weights + (strategy, regime, market) context, and asked to produce one calibrated meta opinion (action + confidence + rationale). Lower max_tokens (200) and lower temperature (0.2) for deterministic synthesis. The prior cycle's meta opinion is cached per (strategy:symbol) for 30 min and injected as a prompt block into the NEXT cycle's per-model prompts so voters can see and disagree with the prior take. Strictly informational — meta output is surfaced in the SIGNAL audit row + dashboard but does NOT replace the raw quorum decision and does NOT loosen the gate.
  - `services/llmService.js` — ensemble decision orchestrator. Dynamic-weighting + meta-reasoner integration: STEP 1 resolves per-model weights for the current (strategy, regime, market) context; STEP 2 looks up the prior meta opinion; STEP 3 runs the 4 voters in parallel with both blocks injected; STEP 4 runs the meta-reasoner synthesis. Computes BOTH raw and weighted tallies. **Critical safety contract**: the `consensus` field returned to `riskManager.checkQuorum` remains the RAW unweighted majority vote, and `confidence` returned is `min(rawAvgConfidence, weightedConfidence)` so dynamic weights can ONLY tighten the gate, never relax it. New informational fields exposed for audit/dashboard: `rawConfidence`, `weightedConsensus`, `weightedConfidence`, `weightedVotes`, `weights`, `weightContext`, `meta` — none of these gate trading. Quorum (3-of-4 raw count), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, no-averaging-in, and trailing-stop ratchet ALL retain full veto power.
  - `services/causalInferenceService.js` — causal inference layer. Mines the closed-trade history (joined to TRADE_EXECUTED audit rows) to identify which decision-time features have a *real* lift on win-rate vs spurious correlations. For each (strategy × regime × market) bucket with ≥12 closes, computes per-feature conditional win-rate true vs false, lift over baseline, avg-P&L lift, and a sample-weighted confidence tier (high ≥15pp / medium ≥8pp / low ≥6pp / spurious <6pp). Spurious filter drops edges where |lift| < 6pp even at high N — small lifts at large samples are noise, not signal. Persists per-bucket payloads as JSONB in new `causal_insights` table. Renders compact prompt block listing top 5 positive AND top 5 negative drivers (filtered, sorted by |lift|). 30-min in-memory cache + 30-min refresh cadence (boot warm-up at +90s). Features mined: high_confidence (≥90%), unanimous_quorum (4/4), meta_agrees, meta_high_conf, weighted_matches_raw, rsi_neutral, macd_positive, positive_news, high_volume, options_unusual, ml_high_pwin. Endpoint: `GET /api/causal-insights?strategy=&regime=&market=`. Strictly informational — never gates trading.
  - `services/counterfactualService.js` — counterfactual reasoning layer. Replays decision-rule counterfactuals over recent closes (e.g. "require entry conf ≥ 88%", "require 4/4 unanimous quorum", "skip when meta-reasoner disagreed", "require weighted consensus = raw consensus", "skip VOL_SPIKE/RATE_SHOCK regimes", "require ML p(win) ≥ 55%") and tells you what aggregate P&L would have been if the rule had been live. Pure decision-rule replay — NOT bar-replay — so no historical bar refetch needed; uses only audit metadata available at decision time. Per (strategy × regime × market) bucket with ≥10 closes: computes for each counterfactual the keptN/droppedN, kept-side P&L, dropped-side wins/losses, ΔP&L vs baseline, and verdict tag (IMPROVES if Δ > 10% of |baseline| + $5 tolerance, WORSENS if symmetric below, else wash). Renders 3-5 line summary sorted by |ΔP&L| so the strongest signals come first. 30-min refresh cadence (boot warm-up at +90s). Endpoint: `GET /api/counterfactuals?strategy=&regime=&market=`. Strictly informational — quorum (3-of-4), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, no-averaging-in, and trailing-stop ratchet ALL retain full veto power.
  - `services/feedbackService.js` — Human-in-the-Loop Feedback & Fine-Tuning Layer. Captures user feedback on closed trades (1-5 star rating + optional natural-language comment) and mines it into three calibration signals fed back into the decision loop. **One row per piece of feedback** in new `trade_feedback` table (rating nullable, derived sentiment good/bad/neutral, derived tags from comment via closed whitelist of regex patterns — `too_aggressive`, `too_late`, `wrong_regime`, `good_entry`, `bad_exit`, `stop_too_tight`, etc.). **Three signals derived per (strategy × regime × market) bucket**: (a) **Per-model FEEDBACK BIAS** — for each voting model, signal = (good_votes - bad_votes) / total mapped to multiplier in [0.85, 1.10]; applied to llmWeightingService weights BEFORE renormalisation so post-renorm Σweights still equals N (user feedback can only re-balance among models, never lift the absolute weighted-confidence sum); requires ≥4 samples per model. (b) **CONFIDENCE SHRINKAGE** — if avg rating of HIGH-conf BUYs (≥85%) is meaningfully WORSE than low-conf BUYs in the bucket, the model is overconfident → shrinkage factor in [0.85, 1.00] returned; **ceiling of 1.00 enforces the ONE-WAY safety contract**: this layer can ONLY tighten the gate, never relax it. Composed via `gateConfidence = min(rawAvgConfidence, weightedConfidence) * fbShrink` so all defenses stack rather than override; bad/missing/NaN inputs default to 1.0 (no shrink). Requires ≥3 samples each side and ≥6 total. (c) **PROMPT NUDGE** — pre-rendered "USER FEEDBACK — N rated trades, avg X/5, common tags: too_aggressive ×8" block injected via `feedbackContext` into every voter's prompt so the model sees the human signal. Comments themselves are NEVER injected into prompts — only the closed-whitelist tags + aggregate counts — eliminating prompt-injection / jailbreak risk via stored user text. 5-min calibration cache + in-flight dedupe + cache-bust on new feedback insert. Endpoints: `POST /api/trades/:id/feedback` (USER-facing, NOT operator-gated, body `{rating, comment}`, rejects rating<1 or rating>5 or missing trade), `GET /api/feedback/recent?limit=` (read-only), `GET /api/feedback/calibration` (current per-bucket biases + shrinkage factors). Frontend: `TradeLog.jsx` row inline `FeedbackControl` (5-star tap + 1-line comment, only on SELL+pnl rows) — hydrates already-rated trade ids on mount so re-rating shows ✓. **Critical safety contract**: STRICTLY INFORMATIONAL → INFORMATIONAL TIGHTENING ONLY. Writes only `trade_feedback`. The two numeric outputs (model bias multiplier, confidence shrinkage) are bounded by hard floors/ceilings and composed via existing `min()` gates so they can ONLY make the gate stricter, never relax it. Cannot touch the risk manager, quorum (3-of-4 raw), per-tier confidence floor, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, no-averaging-in rule, or trailing-stop ratchet — ALL retain full veto power. Every failure (DB, mining, render) swallows silently — a feedback hiccup cannot break the trading loop.
  - `services/strategyDiscoveryService.js` — Automated Strategy Discovery Layer. Periodically generates a small population of candidate decision-rule VARIATIONS (confidence-gate sweep at 86/88/90/92%, meta-reasoner-must-agree, meta-conf gate at 70/75/80%, unanimous 4/4 quorum, weighted-must-match-raw, regime-skip filters for VOL_SPIKE/RATE_SHOCK/CHOP, ML p(win) floor at 55/60/65%, plus 4 curated 2-rule combinations — 20 total per cycle), backtests each one as a DECISION-RULE REPLAY over the closed-trade audit history (joined to the originating BUY's TRADE_EXECUTED audit row, identical attribution semantics to counterfactualService), and persists the strongest as PENDING PROPOSALS in new `strategy_proposals` table for the operator to review on the dashboard. Filters: keptN ≥ 10 AND keptN/baselineN ≥ 30% AND deltaPnl ≥ max($5, 5%·|baseline|) per (strategy × regime × market) bucket — guards against over-fitting and "drop-everything" rules. Top 10 per refresh persisted as `status='pending'`. Dedupe: skips insert if same (rule_key, strategy, regime, market) already pending OR applied (dismissed rows can be re-proposed). 4-hour refresh TTL + boot warm-up at +210s + in-flight dedupe. Operator clicks **Apply** → row inserted into new `active_overlays` table (UNIQUE on rule_key+strategy+regime+market), proposal moves to 'applied'; operator clicks **Dismiss** → proposal moves to 'dismissed' with no overlay; operator clicks **Revert** on an active overlay → DELETE from active_overlays + source proposal moves to 'dismissed'. At decision time, riskManager.evaluateBuy AFTER quorum check passes calls `checkOverlays({signal, strategy: sc.name, regime, market})` (lazy-required, fail-safe to allow on any error so a discovery hiccup cannot break the loop) — predicates rehydrated from JSON `def` via `_predicateFromDef` switch (conf_gte / meta_must_agree / meta_conf_gte / unanimous_quorum / weighted_must_match / skip_regime / ml_pwin_gte / combined). Predicates are pure functions of decision-time signal/audit metadata (confidence, votes, meta_opinion, weighted_consensus, regime, ml_pwin) — same shape counterfactualService consumes. Scope match supports '*' wildcard for strategy/regime/market. 30s in-memory overlay cache, busted on apply/revoke. Endpoints: `GET /api/strategy-proposals?status=pending|applied|dismissed`, `GET /api/strategy-discovery/summary` (public reads), `POST /api/strategy-proposals/:id/apply`, `POST /api/strategy-proposals/:id/dismiss`, `DELETE /api/strategy-overlays/:id`, `POST /api/strategy-discovery/refresh` (all operator-gated). Frontend: `StrategyProposalsPanel.jsx` mounted under SafetySuggestionsPanel on the home tab — shows pending proposals with delta-P&L, baseline vs kept stats (n, win rate, P&L) and one-tap Apply/Dismiss; active overlays section with Revert. Polls every 90s. **Critical safety contract — STRICTLY ADDITIVE FILTERING**: discovery NEVER auto-applies (only persists 'pending' rows); apply/dismiss/revert all require operator token; `checkOverlays()` can ONLY return `allow:false` (downgrade BUY→HOLD via the same `{allow:false, reason}` path the existing gate uses) — there is no code path that returns `allow:true` when the existing stack said no, so overlays can NEVER lift confidence, override quorum, expand sizing, bypass the daily loss cap / 5% drawdown breaker / kill switch / no-averaging-in rule / trailing-stop ratchet. Predicate failures swallow to don't-filter (cold-start safe), DB failures fail-safe to no-overlays (existing safety stack remains fully in force). Variation predicates and rehydration are pure functions — no external I/O at gate-check time.
  - `services/propagationService.js` — Cross-Market & Sector Propagation Layer. Learns how moves in one (market × sector) propagate to another (e.g. "US Technology bearish → ASX Materials trades win 38% vs 56% baseline, -18pp"). Curated 57-symbol sector taxonomy bucketed into coarse groups (Technology / Financials / Materials / Healthcare / Energy / Consumer / Communication / Industrials / RealEstate / ETF). Per refresh (30 min cadence + boot warm-up at +180s): (1) computes CURRENT per-bucket "pulse" from last-24h SIGNAL audit rows aggregating signal direction + news polarity + recent-bar return, bucketed into bullish/bearish/neutral via ±0.10 thresholds (requires ≥3 signals per bucket to call a state); (2) mines propagation edges from closed SELL trades by looking up what every other (market, sector) pulse was in the 24h preceding entry, grouped by (target × source × source_state), keeps edges with n_samples ≥ 5 AND |lift_pp| ≥ 8 vs the target bucket's own baseline win-rate. Persisted to new `propagation_insights` table with partial UNIQUE index on (target_market, target_sector, source_market, source_sector, source_state). Mining wipe-and-rewrite runs under one DB transaction (DELETE + INSERT) so retrieval never sees a half-empty table. In-flight dedupe + 30-min throttle (force=true bypasses for boot warm-up). Hour-keyed in-memory pulseCache during mining collapses redundant SQL hits within the same hour, so the inner loop is O(unique_hours × buckets) not O(trades × buckets). At decision time, agent calls `renderForPrompt(symbol)` which looks up the symbol's (market, sector), finds top-3 propagation edges where the source bucket is CURRENTLY in the conditioning state (mismatch → edge silently inactive), renders compact `CROSS-MARKET PROPAGATION — N active edges affecting US Technology:` block with TAILWIND/HEADWIND tags, ending with explicit reminder "Treat as priors, NOT as gates. The 3-of-4 quorum + confidence gate retain full veto power." Self-edges (target == source) skipped during mining. **Critical safety contract**: strictly informational prompt-injection layer. Writes only `propagation_insights`. Never touches the risk manager, never gates trades, never alters quorum/confidence/breakers/loss-cap/trailing-stops/kill-switch. Every failure (DB, mining, render) swallows silently — a propagation hiccup cannot break the trading loop. Endpoints: `GET /api/propagation` (read-only summary: current pulses + top edges), `POST /api/propagation/refresh` (operator-gated force re-mine).
  - `services/memoryService.js` — Long-Term Memory & Experience Replay Layer. One row per closed trade in `trade_memory` (full structured context + outcome + 1-line auto-generated lesson + fixed-length 28-dim normalized `feature_vec` + `feature_schema_version` for forward-compat). Uses a structured feature-vector embedding rather than text embeddings (no API call, deterministic, more meaningful for tabular trading contexts; pgvector NOT required). Per-cycle: featurises CURRENT (strategy × regime × market × indicator buckets × news polarity × patterns) decision context, runs in-memory cosine similarity against the last N=2000 memories, takes top-5 with sim ≥ 0.6, balances wins/losses from a wider top-15 candidate pool to avoid one-sided priors, renders a compact "EXPERIENCE REPLAY — N similar past situations retrieved (W/L, agg P&L)" block ending with the explicit reminder "Use these as priors, NOT as gates. The 3-of-4 quorum + confidence gate retain full veto power.". Block is injected via `experienceContext` in `llmService.buildPrompt` alongside causal/counterfactual blocks. Backfill scans closed SELL trades not yet indexed, joins to nearest-prior SIGNAL audit row + matching BUY trade for entry context. Boot warm-up at +150s + 30-min refresh interval. In-flight dedupe on both backfill and cache load; backfill forces a fresh cache reload after inserts (resets `_cacheLoadedAt` and `_cacheInFlight`) to prevent adopting stale pre-insert reads. Regime input normalized via `_regimeKey()` accepting either string or `regimeService.classifyRegime()` object (extracts `.primary`). **Critical safety contract**: strictly informational prompt-injection layer. Writes only `trade_memory`. Never touches risk manager, never gates trades, never alters quorum/gate/sizing/breakers/kill switch/trailing-stops/loss-cap. Every failure (DB, retrieval, render) swallows silently — a memory hiccup cannot break the trading loop. Endpoints: `GET /api/memory?strategy=&regime=&market=&limit=` (read-only summary + sample), `POST /api/memory/refresh` (operator-gated force backfill).
  - `services/safetySuggestionService.js` — Intelligent Safety Suggestion Layer. Analyses recent realised P&L (last 30 global closes + per-strategy 15 closes) plus counterfactual evidence, generates a small bounded set of data-backed safety recommendations. Two whitelisted suggestion kinds: `risk_scale_change` (step DOWN to conservative when net P&L < -$50 OR wr < 40% OR ≥2 contexts where tightening would have improved P&L; step UP from conservative to balanced only when net > +$50 AND wr > 60% — never auto-suggests aggressive) and `strategy_disable` (pause an enabled strategy when its last 15 closes show wr < 35% AND net < -$30). Persists to new `safety_suggestions` table with status pending → applied | rejected | expired (24h auto-expiry). Partial unique index on (kind, target, suggested_value) WHERE status='pending' so re-mining updates rationale rather than piling duplicates. **Critical safety contract**: NOTHING is ever applied automatically — every transition requires an explicit user POST. The applier dispatches via a hard-coded `KINDS` whitelist; an unknown kind is refused before any state writer is called. Suggestions can ONLY invoke existing audited writers (`setRiskScale`, `setStrategyEnabled`) — i.e. things the user could already do themselves through the dashboard. We never invent new tunable surfaces, never expose the 3-of-4 quorum, the 5% drawdown circuit breaker, the kill switch, the trailing-stop ratchet, or the per-tier confidence/loss-cap floors as knobs. Both apply and reject transitions write tamper-evident audit rows (SUGGESTION_APPLIED / SUGGESTION_REJECTED). Throttle + in-flight dedupe (30-min TTL, module-load timestamp init, force flag for boot warm-up). Endpoints: `GET /api/safety-suggestions` (pending + summary), `GET /api/safety-suggestions/recent?limit=` (history), `POST /api/safety-suggestions/refresh` (operator-triggered re-mine), `POST /api/safety-suggestions/:id/apply`, `POST /api/safety-suggestions/:id/reject`. Frontend: `SafetySuggestionsPanel.jsx` on the Home tab — lists pending suggestions sorted by severity (high → medium → low), each card has Apply / Dismiss buttons, Apply prompts for confirmation, severity-coloured borders (red/yellow/blue), shows rationale + evidence line + hours-until-expiry, polls `/api/safety-suggestions` every 60s, manual "↻ Re-mine" button. Empty state shows "No safety suggestions right now — all metrics within healthy bands." Boot warm-up at +120s (after counterfactual cache populated) + 30-min refresh interval.
  - `services/optionsFlowService.js` — quantitative options-chain analytics. Pulls Alpaca's `/v1beta1/options/snapshots/{underlying}` (paginated, capped at 200 contracts within ±15% of spot and ≤60 DTE). Computes: P/C volume ratio, P/C OI ratio, ATM IV (mean of contracts within 2.5% of spot), IV skew (5%-OTM put IV − 5%-OTM call IV at nearest expiry), IV rank 0–100 over rolling 252-day window from `iv_history` table, and unusual-activity contracts (sweep proxy: vol/OI ≥ 2 AND vol ≥ 100; block proxy: vol ≥ 1000). 30-min in-memory TTL + per-symbol in-flight dedup. Refresh batch runs 60s after boot and every 30 min during US market hours; spot prices for the strike filter come from the latest 1-min bar (`buildSpotLookup`). Injected into every LLM prompt as the `optionsFlow` block (1–4 lines: P/C ratio, IV/IVR/skew, sentiment hint, top-3 unusual contracts) — strictly informational, never affects quorum/gate/sizing/breaker. US-only (Alpaca options coverage). Endpoint: `GET /api/options-flow/:symbol`.
  - `services/rlExecutionService.js` — reinforcement learning execution layer. Tabular Q-learning over (state, action) cells in `rl_q_table`. State = `regime|strategy|mfe-bucket|pnl-bucket` (~375 cells max). Actions: NONE, TIGHTEN (trail × 0.7), LOOSEN (trail × 1.4), ARM_EARLY (activate × 0.5), LOCK_IN (trail = 0.7%, arm immediately at +0.01% profit). ε-greedy (decays from 0.10 to 0.02 over first 200 visits per state), α=0.20. Per cycle, `recommendForHolding` picks an action and returns an `adjustedConfig` with `trailingStopPct` / `trailingActivatePct` clamped to [0.5×, 1.5×] of the strategy default — fed straight into the existing `riskManager.computeTrailingUpdate`, which still only ratchets stops UP. On every closed trade, `recordOutcome` pulls the symbol/strategy's RL_EXEC_DECISION audit rows since the most-recent BUY, computes R-multiple = `pnl/risk` (clipped ±3, risk pulled from BUY's `ml_risk_usd`), groups by (state, action) and applies a weighted SGD-equivalent update `Q ← Q + (1−(1−α)^w)·(R−Q)`. Audit logging is debounced — same-action skips coalesce up to 10 cycles. All errors swallowed; quorum, confidence gate, daily loss budget, % drawdown breaker, kill switch are completely untouched. Endpoint: `GET /api/rl/execution`.
  - `services/knowledgeGraphService.js` — long-term per-symbol company knowledge graph (Postgres `company_knowledge`). Stores company metadata, sector + peers, rolling earnings track (≤6), valuation, macro context, and a curated major-event timeline (≤6) extracted best-effort by Grok-mini. Daily refresh at 08:00 UTC + 30s post-boot warm-up; event-triggered staleness when sentiment |score|≥0.5. Two-phase refresh persists deterministic core/summary BEFORE optional LLM enrichment, so the prompt block is never gated on LLM latency. Hard 1400-char summary cap. Injected as `knowledgeContext` in the LLM ensemble — informational only, never affects quorum/gate/sizing. Endpoints: `GET /api/knowledge`, `GET /api/knowledge/:symbol`.
  - `services/backtestService.js` — daily-bars rules-based backtest engine (RSI + trend + MACD entry, configurable stop/target/trailing). Models slippage (bps) and per-trade commission. Returns equity curve, trade log, Sharpe, max drawdown, win rate. Persists to `backtest_runs` table. Watchlist-only, in-flight lock, operator-gated.
- **Frontend** (`frontend/`, port 5000): React + Vite + Tailwind v4. iPhone-16 glassmorphism dashboard. Tabs: Home, Markets, Strategies, Reasoning, Positions, Trades, **Backtest**, Settings. Tooltips on every button. Strategy ON/OFF toggles, paper↔live mode switcher with confirmation modal. Voice broker (`VoiceChat` + `useVoice` hook) via Web Speech API. Markets tab shows a clean recharts price chart per watchlist symbol (1d / 5d toggle) with live AI confidence, news sentiment score, and key insights.
- **Database**: PostgreSQL — `portfolio`, `holdings` (composite PK), `trades`, `audit_log`, `historical_intelligence`, `symbol_strategy_performance`, `model_performance`, `backtest_runs`, `company_knowledge`, `rl_q_table`, `iv_history`.

## Multi-Market & FX
- One unified USD-equivalent virtual portfolio: ASX cash debits/credits = `native_price × qty × fx_at_trade`. Holdings store NATIVE qty + avg_cost + currency + fx_rate_at_entry; equity is summed in USD using a USD-equivalent price lookup so the **single $100/day USD loss budget and 5% drawdown circuit breaker apply across BOTH markets**.
- Risk sizing in USD-equivalent: `latest_native × fxToUsd` is fed to `evaluateBuy`; returned stop/take are USD-equivalent and converted back to native for both the broker order and `holdings` storage.
- Per-strategy market gating in `runCycle`: cycle proceeds when ANY market is open (so equity stays mark-to-market for the breaker), but each strategy only places new orders when its own market is open.
- Operator emergency controls fan out across brokers: kill-switch, cancel-all-orders, and flatten-all all hit BOTH Alpaca and IBKR via `brokerRouter.*AllBrokers` helpers (allSettled — one broker failing doesn't skip the other).
- US-only enrichment services (premarket briefing, intraday tactical, 20-yr historical intelligence, options activity) are skipped for ASX symbols — analyzer falls back to bar-derived signals (the safe degradation for a new market).

## Safety Features
- Server-side 85% confidence gate (cannot be bypassed by frontend).
- Circuit breaker auto-trips at 5% daily drawdown; requires manual reset.
- Emergency pause button halts all trading immediately.
- ≥3/4 LLMs must respond or system forces HOLD.
- Stop-loss skipped on stale price data (no fallback to avg cost).
- All decisions, trades, rejections, errors logged to `audit_log`.
- Allowlisted SQL column writes prevent injection via `updatePortfolio`.
- Atomic cash updates via `UPDATE ... SET cash = cash + delta` (no read-modify-write race).
- **Kill switch (Phase 4)**: `POST /api/agent/kill-switch` (operator-token + body `{confirm:"KILL"}`) trips a sticky in-memory flag set BEFORE any await, then cancels open orders, drains in-flight cycle (30s timeout), flattens all positions. Once tripped, `executeOrder()`, `runCycle()`, auto-hedge, AND `alpacaService.placeOrder()` itself all refuse new orders; `/api/agent/start` and `/run-now` return 409 "kill switch latched — restart required". Only a process restart clears it.
- **Cancel all open orders (Phase 4)**: `POST /api/agent/cancel-orders` audit-logged broker order cancellation, separate from flatten.
- **Audit hash-chain (Phase 4)**: `audit_log` has `prev_hash` + `row_hash` (SHA-256 of `prev_hash || event_type || symbol || decision || JSON(payload) || created_at`), written inside a transaction with `LOCK TABLE audit_log IN EXCLUSIVE MODE`. `GET /api/audit/verify` walks the chain and reports the first break (or "ok"). Tamper-evident — any row mutation invalidates every subsequent `row_hash`.
- **Compliance reports (Phase 4)**: `GET /api/audit/report?date=YYYY-MM-DD&format=json|csv` (operator-token gated). JSON includes daily P&L, win rate, model attribution, risk events (kill switch / circuit breaker / hedge / mode changes / drawdown pause), blocked-trade reasons (quorum/gate/risk), per-trade SIGNAL attachment via two-pass strategy-aware match (strict strategy === trade.strategy preferred, falls back to any-strategy within 5-min window), hash-chain verification result. CSV is a flat regulator-relevant trade sheet.
- **Asset-class scaffolding (Phase 4)**: `trades`, `holdings`, `audit_log` have `asset_class` column (default `'equity'`). Equities-only execution today; options/futures execution intentionally deferred (would require LLM prompt restructure, contracts×multiplier sizing, different stop-loss math, and Alpaca's separate options contract symbol schema — futures not supported by Alpaca at all).

## US/ASX Market Split (May 2026)

The dashboard now treats US and ASX as first-class peers everywhere:

- **Header:** two market badges (🇺🇸 / 🇦🇺 OPEN/CLOSED) plus a compact AUD/USD FX pill (green=live, yellow=stale, red=missing). Stale rates make ASX risk-sizing refuse — operators see this at a glance.
- **Home / MarketsTab:** `MarketClocks` renders the NYSE and ASX (Sydney) sessions side-by-side with DST-correct countdowns via `Intl`. Full `FxBadge` panel sits below.
- **Per-tab market filters** (`MarketFilter` chips, `ALL/US/ASX` with live counts) on Live Signals, Reasoning, Positions, Trades, and Backtest. Each filter has its own state so scoping one section never narrows another.
- **Holdings** are shown in NATIVE currency per row ($/A$ + market badge). Positions tab adds per-market subtotals (native + USD-equivalent via `state.fx`) so AUD never gets summed into USD.
- **Trades + Audit** carry `market`/`currency`/`fx_rate`. The TradeLog displays the FX rate used at execution for ASX rows. `/api/trades` and `/api/audit` accept `?market=US|ASX`; platform-wide audit events without a symbol stay visible under any filter.
- **Backtest** scope chip narrows the allowed watchlist to one market and auto-swaps default symbols (US: SPY,QQQ,AAPL; ASX: BHP,CBA,CSL). Recent runs are tagged US/ASX/MIX and filterable by chip.
- **Backend tagging:** `/api/markets` returns combined `usWatchlist`+`asxWatchlist` and tags each card with `market`/`currency`. `/api/bars/:symbol` and `/api/sentiment/:symbol` whitelist both watchlists and route ASX through `brokerRouter` (IBKR). `agent.js` writes `market`+`currency` onto every `lastSignals` entry.

All existing safety rules (single USD-equivalent daily loss budget, circuit breaker, kill switch, dual-broker order routing) are unchanged — only the presentation and filtering layers were extended.
