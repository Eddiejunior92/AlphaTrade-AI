# AlphaTrade AI v2

Production-ready autonomous multi-LLM high-frequency trading agent.

## Architecture
- **Backend** (`backend/`, port 3001): Node/Express + WebSocket (`/ws`).
  - `server.js` ŌĆö REST API + WS broadcaster. Operator-token auth on all mutation endpoints; LIVE-mode switch refused entirely if `OPERATOR_TOKEN` is unset.
  - `strategies.js` ŌĆö registry of strategies (day, swing) with their cadences, stops/targets, risk, and overnight rules.
  - `agent.js` ŌĆö base 60s cycle; runs each enabled strategy on its own cadence (day=60s/1Min, swing=300s/15Min). Global trading lock prevents cycle/flatten/mode-switch from overlapping. Day strategy auto-flattens 5m before close; swing holds overnight. Daily reset at 13:30 UTC, auto-resume on restart.
  - `services/llmService.js` ŌĆö 4-model ensemble: Gemini 2.0 Flash, Claude 3.7 Sonnet, GPT-4o (via OpenRouter) + Grok 4 Fast (xAI direct). Requires Ōēź3/4 valid responses.
  - `services/riskManager.js` ŌĆö strategy-aware: takes a `strategyConfig` per evaluation. 85% confidence + 3-of-4 quorum hard gates. Refuses averaging-in (no re-buying same symbol in same strategy). Cross-strategy mark-to-market for circuit-breaker math. $100/day USD loss budget. **Compounding Confidence-Weighted sizing** (`computeDynamicScaling` + `computeTargetRisk`): per-trade $ risk = `lerp(min, max, confFraction) ├Ś growthMult ├Ś perfMult`, hard-capped at 2├Ś scale.maxRiskUSD AND `maxPositionPct ├Ś equity` AND cash. `growthMult` steps +5% per +10% equity (range 0.5ŌĆō2.0├Ś). `perfMult` from last-20 closed-trade net PnL as % of starting balance (range 0.8ŌĆō1.2├Ś). `confFraction` lerps signal confidence within `[threshold, 1.0]`.
  - `services/alpacaService.js` ŌĆö runtime paper/live switching via `setMode()`. Live mode reads separate `ALPACA_LIVE_API_KEY` / `ALPACA_LIVE_SECRET_KEY`; refuses live without them.
  - `services/ibkrService.js` ŌĆö IBKR Client Portal Web API broker for ASX symbols. Mock mode when `IBKR_BASE_URL`/`IBKR_ACCOUNT_ID` unset (matches Alpaca's mock pattern). Same kill-switch hook (`setKillSwitchActive`) as Alpaca so the operator kill cascade halts ASX too.
  - `services/brokerRouter.js` ŌĆö per-symbol broker routing (USŌåÆAlpaca, ASXŌåÆIBKR via `marketRegistry`). Cross-broker `closeAllPositionsAllBrokers`, `cancelAllOpenOrdersAllBrokers`, `setKillSwitchActiveAll` so operator emergency controls fan out to both. `getBars` is broker-agnostic ŌĆö bars are normalized to `{t,o,h,l,c,v}` by each service.
  - `services/marketRegistry.js` ŌĆö market metadata for every symbol. ASX watchlist (10 liquid blue-chips: CBA, BHP, CSL, MQG, WES, RIO, FMG, TLS, WOW, ANZ). `isAsxOpen()` and `nextAsxOpen()` use Sydney timezone math (MonŌĆōFri, 10:00ŌĆō16:00 AEST/AEDT ŌĆö no holiday calendar; ASX-holiday orders fail at the broker and are audit-tagged).
  - `services/fxService.js` ŌĆö cached AUDŌåÆUSD rate. Source order: `AUDUSD_RATE` env override ŌåÆ `exchangerate.host` (1h TTL) ŌåÆ hardcoded fallback `0.66`. Returns `{audusd, source, stale, fetchedAt}` for the dashboard so operators can see when FX is stale or has fallen back.
  - `services/brokerService.js` ŌĆö "Alpha" personal-broker chat (Claude 3.7 via OpenRouter), strategy/mode-aware.
  - `services/db.js` ŌĆö PostgreSQL with `ensureSchema()` migration. `holdings` has composite PK `(symbol, strategy)` so the same symbol can be held in both strategies independently.
  - `services/discordService.js` ŌĆö webhook alerts.
  - `services/sentimentService.js` ŌĆö Grok-powered news sentiment per symbol. Returns `{score: -1..+1, label, summary, insights, sources}`. Cached 30 min per symbol; refreshed in background each cycle. Injected into the LLM ensemble prompt and exposed to the dashboard.
  - `services/adaptiveLearningService.js` ŌĆö nightly recompute (22:00 UTC) of per-(symbol,strategy) and per-(model,strategy) win-rate from closed trades. Exposes `getCalibrationHints()` (prompt block, advisory only) and `getSizingMultiplier()` Ōłł [0.7, 1.2]. Never relaxes quorum/gate.
  - `services/portfolioOptimizationService.js` ŌĆö correlation-aware portfolio risk. `evaluateAddition()` returns `sizeMult` Ōłł [0.5, 1.0] based on existing-holding correlation + concentration. Pure SIZING modifier, applied AFTER quorum/gate.
  - `services/hedgingService.js` ŌĆö portfolio-level risk monitor. Posts Discord alert on risk spike; if `AUTO_HEDGE=true`, places SH (inverse SPY) buy at 15% of long exposure. Cooldown-protected, never bypasses circuit breaker. Default OFF (advisory).
  - `services/orderFlowService.js` ŌĆö bar-derived buy/sell pressure proxy (volume-weighted up vs down candles). Lightweight prompt block.
  - `services/optionsActivityService.js` ŌĆö Grok-fetched unusual options activity flag per symbol. Cached, refreshed every 30 min during market hours. Prompt block only ŌĆö never sizes trades.
  - `services/macroFactorService.js` ŌĆö cross-asset macro factor reader. Pulls 12 ETF proxies (SHY/IEF/TLT for the rate curve, TIP for inflation, VXX for vol, UUP for USD, GLD/USO/DBC for commodities, HYG for credit, SPY/EEM for equity risk) via Alpaca's free daily-bar feed (no extra API keys). Computes 1d/5d/20d log-returns + 20d realized vol per factor, then derives composites: yield-curve change (TLT-SHY 5d), breakeven inflation (TIP-IEF 20d), credit-equity divergence (HYG-SPY 5d), and a risk-on/off composite (avg of equity/EM/credit minus avg of VXX/USD/gold, 5d). 60-min in-memory TTL with in-flight dedup. Read-only ŌĆö failures degrade to nulls.
  - `services/macroForecastService.js` ŌĆö forward-looking macro regime forecast layer. Classifies BOTH the current macro regime AND the most likely regime over the next 24-48h with a confidence score, picking from {VOL_SPIKE, RATE_SHOCK, RISK_OFF, STAGFLATION, INFLATION_RISING, USD_STRONG, RISK_ON, GOLDILOCKS, NEUTRAL}. Tiebreak prioritises safety regimes (vol/rate/risk-off > constructive). Forecast is momentum-based: 5d-and-20d aligned signals project forward with higher confidence; 5d-only signals get downgraded with a "potential mean-reversion" flag. Derives safety-preserving adjustments per regime: `confidenceBoost Ōłł [0, 0.05]` (added to the strategy's confidence threshold inside `riskManager.checkQuorum`, additive with `metaLearningService`'s regimeBoost ŌĆö pure tightening) and `sizeMult Ōłł [0.70, 1.00]` (multiplied alongside the existing mults inside `riskManager.evaluateBuy` ŌĆö **hard ceiling of 1.0, macro can NEVER amplify size**). 30-min cache + cold-start identity. Refreshed every 60 min during US market hours after a 75s post-boot warm-up. Injected as the `macroForecast` block in every LLM prompt (~3-5 lines: now/forecast labels with confidences, brief reasoning, risk-posture summary, advisory voice hint). Applies to BOTH US and ASX symbols (cross-asset signals are global). Strictly informational at the LLM layer; quorum (3-of-4), confidence gate, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, and trailing-stop ratchet are ALL untouched and retain full veto power. Endpoint: `GET /api/macro-forecast`.
  - `services/scenarioSimService.js` ŌĆö self-play Monte-Carlo scenario simulation. Pure-JS GBM path generator (configurable via `SIM_PATHS`, default 400 paths ├Ś 3 trading-day horizon) that fuses recent bars + ATR% + IV (when present) + regime + macro forecast into a probability-weighted forward outlook. Vol blends three sources (realized per-bar ŌåÆ daily, ATR%, IV/ŌłÜ252) with adaptive weights (IV gets 1.2├Ś since it's forward-looking) then multiplied by a regime-conditional vol multiplier Ōłł [0.8, 1.6] (high-vol/VOL_SPIKE/RATE_SHOCK expand tails; GOLDILOCKS dampens). Drift = OLS log-slope of last 30 closes per-day, capped at ┬▒0.4%/day, plus regime+macro tilt (asymmetric ŌĆö adverse regimes pull harder than constructive ones lift). Optional jump-diffusion overlay during stress regimes (10%/day chance of ┬▒2-3% Laplace-tail jump, biased negative in RISK_OFF/STAGFLATION). Computes per-day median/5th/95th percentiles, P(upŌēź1%/2%), P(downŌēź1%/2%), P(strategy stop hit), P(strategy target hit), and 3d expected return ŌĆö all derived from the actual path bundle. 3-min cache keyed on lastBarT so re-runs in the same minute hit cache. Renders 3-5 line prompt block injected as the `scenarioSim` block in every LLM prompt (both day + swing). Strictly informational ŌĆö does NOT vote, size, or gate; quorum (3-of-4), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, and trailing-stop ratchet ALL retain full veto power. Endpoint: `GET /api/scenario-sim/:symbol`.
  - `services/llmWeightingService.js` ŌĆö dynamic LLM ensemble weighting layer. Learns optimal per-model weights per (strategy ├Ś regime ├Ś market) from realised P&L attributed to the originating BUY's voting models. New `model_regime_performance` table stores the finer-grained context buckets alongside the existing (model ├Ś strategy) `model_performance` table. Resolution order per model: full-trust on the (model, strategy, regime, market) bucket when n Ōēź 12, else linear blend with the (model, strategy) bucket weighted by sample-count vs trust floor, else strategy-only, else uniform 1.0 (cold-start). Bayesian-smoothed win-rate (prior n=8 trades at 50% wr) ŌåÆ weight Ōłł [0.5, 1.5], renormalised so ╬Żweights = model count (keeps weighted vote tally on the same numeric scale as raw count). 60-second cache, busted on every recordOutcome write so the very next cycle sees fresh weights. Wired into `adaptiveLearningService.recordOutcome` to update on every closed trade with regime + market resolved from the originating TRADE_EXECUTED audit row (best-effort, never blocks). Endpoint: `GET /api/llm-weights?strategy=&regime=&market=`.
  - `services/metaReasonerService.js` ŌĆö final meta-reasoner synthesis layer. After the 4 ensemble models vote, runs ONE additional Claude 3.7 Sonnet call (configurable via `META_REASONER_MODEL`) that's given the raw votes + dynamic ensemble weights + (strategy, regime, market) context, and asked to produce one calibrated meta opinion (action + confidence + rationale). Lower max_tokens (200) and lower temperature (0.2) for deterministic synthesis. The prior cycle's meta opinion is cached per (strategy:symbol) for 30 min and injected as a prompt block into the NEXT cycle's per-model prompts so voters can see and disagree with the prior take. Strictly informational ŌĆö meta output is surfaced in the SIGNAL audit row + dashboard but does NOT replace the raw quorum decision and does NOT loosen the gate.
  - `services/llmService.js` ŌĆö ensemble decision orchestrator. Dynamic-weighting + meta-reasoner integration: STEP 1 resolves per-model weights for the current (strategy, regime, market) context; STEP 2 looks up the prior meta opinion; STEP 3 runs the 4 voters in parallel with both blocks injected; STEP 4 runs the meta-reasoner synthesis. Computes BOTH raw and weighted tallies. **Critical safety contract**: the `consensus` field returned to `riskManager.checkQuorum` remains the RAW unweighted majority vote, and `confidence` returned is `min(rawAvgConfidence, weightedConfidence)` so dynamic weights can ONLY tighten the gate, never relax it. New informational fields exposed for audit/dashboard: `rawConfidence`, `weightedConsensus`, `weightedConfidence`, `weightedVotes`, `weights`, `weightContext`, `meta` ŌĆö none of these gate trading. Quorum (3-of-4 raw count), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, no-averaging-in, and trailing-stop ratchet ALL retain full veto power.
  - `services/causalInferenceService.js` ŌĆö causal inference layer. Mines the closed-trade history (joined to TRADE_EXECUTED audit rows) to identify which decision-time features have a *real* lift on win-rate vs spurious correlations. For each (strategy ├Ś regime ├Ś market) bucket with Ōēź12 closes, computes per-feature conditional win-rate true vs false, lift over baseline, avg-P&L lift, and a sample-weighted confidence tier (high Ōēź15pp / medium Ōēź8pp / low Ōēź6pp / spurious <6pp). Spurious filter drops edges where |lift| < 6pp even at high N ŌĆö small lifts at large samples are noise, not signal. Persists per-bucket payloads as JSONB in new `causal_insights` table. Renders compact prompt block listing top 5 positive AND top 5 negative drivers (filtered, sorted by |lift|). 30-min in-memory cache + 30-min refresh cadence (boot warm-up at +90s). Features mined: high_confidence (Ōēź90%), unanimous_quorum (4/4), meta_agrees, meta_high_conf, weighted_matches_raw, rsi_neutral, macd_positive, positive_news, high_volume, options_unusual, ml_high_pwin. Endpoint: `GET /api/causal-insights?strategy=&regime=&market=`. Strictly informational ŌĆö never gates trading.
  - `services/counterfactualService.js` ŌĆö counterfactual reasoning layer. Replays decision-rule counterfactuals over recent closes (e.g. "require entry conf Ōēź 88%", "require 4/4 unanimous quorum", "skip when meta-reasoner disagreed", "require weighted consensus = raw consensus", "skip VOL_SPIKE/RATE_SHOCK regimes", "require ML p(win) Ōēź 55%") and tells you what aggregate P&L would have been if the rule had been live. Pure decision-rule replay ŌĆö NOT bar-replay ŌĆö so no historical bar refetch needed; uses only audit metadata available at decision time. Per (strategy ├Ś regime ├Ś market) bucket with Ōēź10 closes: computes for each counterfactual the keptN/droppedN, kept-side P&L, dropped-side wins/losses, ╬öP&L vs baseline, and verdict tag (IMPROVES if ╬ö > 10% of |baseline| + $5 tolerance, WORSENS if symmetric below, else wash). Renders 3-5 line summary sorted by |╬öP&L| so the strongest signals come first. 30-min refresh cadence (boot warm-up at +90s). Endpoint: `GET /api/counterfactuals?strategy=&regime=&market=`. Strictly informational ŌĆö quorum (3-of-4), 85% confidence threshold, $100/day USD loss budget, 5% drawdown circuit breaker, kill switch, no-averaging-in, and trailing-stop ratchet ALL retain full veto power.
  - `services/memoryService.js` ŌĆö Long-Term Memory & Experience Replay Layer. One row per closed trade in `trade_memory` (full structured context + outcome + 1-line auto-generated lesson + fixed-length 28-dim normalized `feature_vec` + `feature_schema_version` for forward-compat). Uses a structured feature-vector embedding rather than text embeddings (no API call, deterministic, more meaningful for tabular trading contexts; pgvector NOT required). Per-cycle: featurises CURRENT (strategy ├Ś regime ├Ś market ├Ś indicator buckets ├Ś news polarity ├Ś patterns) decision context, runs in-memory cosine similarity against the last N=2000 memories, takes top-5 with sim Ōēź 0.6, balances wins/losses from a wider top-15 candidate pool to avoid one-sided priors, renders a compact "EXPERIENCE REPLAY ŌĆö N similar past situations retrieved (W/L, agg P&L)" block ending with the explicit reminder "Use these as priors, NOT as gates. The 3-of-4 quorum + confidence gate retain full veto power.". Block is injected via `experienceContext` in `llmService.buildPrompt` alongside causal/counterfactual blocks. Backfill scans closed SELL trades not yet indexed, joins to nearest-prior SIGNAL audit row + matching BUY trade for entry context. Boot warm-up at +150s + 30-min refresh interval. In-flight dedupe on both backfill and cache load; backfill forces a fresh cache reload after inserts (resets `_cacheLoadedAt` and `_cacheInFlight`) to prevent adopting stale pre-insert reads. Regime input normalized via `_regimeKey()` accepting either string or `regimeService.classifyRegime()` object (extracts `.primary`). **Critical safety contract**: strictly informational prompt-injection layer. Writes only `trade_memory`. Never touches risk manager, never gates trades, never alters quorum/gate/sizing/breakers/kill switch/trailing-stops/loss-cap. Every failure (DB, retrieval, render) swallows silently ŌĆö a memory hiccup cannot break the trading loop. Endpoints: `GET /api/memory?strategy=&regime=&market=&limit=` (read-only summary + sample), `POST /api/memory/refresh` (operator-gated force backfill).
  - `services/safetySuggestionService.js` ŌĆö Intelligent Safety Suggestion Layer. Analyses recent realised P&L (last 30 global closes + per-strategy 15 closes) plus counterfactual evidence, generates a small bounded set of data-backed safety recommendations. Two whitelisted suggestion kinds: `risk_scale_change` (step DOWN to conservative when net P&L < -$50 OR wr < 40% OR Ōēź2 contexts where tightening would have improved P&L; step UP from conservative to balanced only when net > +$50 AND wr > 60% ŌĆö never auto-suggests aggressive) and `strategy_disable` (pause an enabled strategy when its last 15 closes show wr < 35% AND net < -$30). Persists to new `safety_suggestions` table with status pending ŌåÆ applied | rejected | expired (24h auto-expiry). Partial unique index on (kind, target, suggested_value) WHERE status='pending' so re-mining updates rationale rather than piling duplicates. **Critical safety contract**: NOTHING is ever applied automatically ŌĆö every transition requires an explicit user POST. The applier dispatches via a hard-coded `KINDS` whitelist; an unknown kind is refused before any state writer is called. Suggestions can ONLY invoke existing audited writers (`setRiskScale`, `setStrategyEnabled`) ŌĆö i.e. things the user could already do themselves through the dashboard. We never invent new tunable surfaces, never expose the 3-of-4 quorum, the 5% drawdown circuit breaker, the kill switch, the trailing-stop ratchet, or the per-tier confidence/loss-cap floors as knobs. Both apply and reject transitions write tamper-evident audit rows (SUGGESTION_APPLIED / SUGGESTION_REJECTED). Throttle + in-flight dedupe (30-min TTL, module-load timestamp init, force flag for boot warm-up). Endpoints: `GET /api/safety-suggestions` (pending + summary), `GET /api/safety-suggestions/recent?limit=` (history), `POST /api/safety-suggestions/refresh` (operator-triggered re-mine), `POST /api/safety-suggestions/:id/apply`, `POST /api/safety-suggestions/:id/reject`. Frontend: `SafetySuggestionsPanel.jsx` on the Home tab ŌĆö lists pending suggestions sorted by severity (high ŌåÆ medium ŌåÆ low), each card has Apply / Dismiss buttons, Apply prompts for confirmation, severity-coloured borders (red/yellow/blue), shows rationale + evidence line + hours-until-expiry, polls `/api/safety-suggestions` every 60s, manual "Ōå╗ Re-mine" button. Empty state shows "No safety suggestions right now ŌĆö all metrics within healthy bands." Boot warm-up at +120s (after counterfactual cache populated) + 30-min refresh interval.
  - `services/optionsFlowService.js` ŌĆö quantitative options-chain analytics. Pulls Alpaca's `/v1beta1/options/snapshots/{underlying}` (paginated, capped at 200 contracts within ┬▒15% of spot and Ōēż60 DTE). Computes: P/C volume ratio, P/C OI ratio, ATM IV (mean of contracts within 2.5% of spot), IV skew (5%-OTM put IV ŌłÆ 5%-OTM call IV at nearest expiry), IV rank 0ŌĆō100 over rolling 252-day window from `iv_history` table, and unusual-activity contracts (sweep proxy: vol/OI Ōēź 2 AND vol Ōēź 100; block proxy: vol Ōēź 1000). 30-min in-memory TTL + per-symbol in-flight dedup. Refresh batch runs 60s after boot and every 30 min during US market hours; spot prices for the strike filter come from the latest 1-min bar (`buildSpotLookup`). Injected into every LLM prompt as the `optionsFlow` block (1ŌĆō4 lines: P/C ratio, IV/IVR/skew, sentiment hint, top-3 unusual contracts) ŌĆö strictly informational, never affects quorum/gate/sizing/breaker. US-only (Alpaca options coverage). Endpoint: `GET /api/options-flow/:symbol`.
  - `services/rlExecutionService.js` ŌĆö reinforcement learning execution layer. Tabular Q-learning over (state, action) cells in `rl_q_table`. State = `regime|strategy|mfe-bucket|pnl-bucket` (~375 cells max). Actions: NONE, TIGHTEN (trail ├Ś 0.7), LOOSEN (trail ├Ś 1.4), ARM_EARLY (activate ├Ś 0.5), LOCK_IN (trail = 0.7%, arm immediately at +0.01% profit). ╬Ą-greedy (decays from 0.10 to 0.02 over first 200 visits per state), ╬▒=0.20. Per cycle, `recommendForHolding` picks an action and returns an `adjustedConfig` with `trailingStopPct` / `trailingActivatePct` clamped to [0.5├Ś, 1.5├Ś] of the strategy default ŌĆö fed straight into the existing `riskManager.computeTrailingUpdate`, which still only ratchets stops UP. On every closed trade, `recordOutcome` pulls the symbol/strategy's RL_EXEC_DECISION audit rows since the most-recent BUY, computes R-multiple = `pnl/risk` (clipped ┬▒3, risk pulled from BUY's `ml_risk_usd`), groups by (state, action) and applies a weighted SGD-equivalent update `Q ŌåÉ Q + (1ŌłÆ(1ŌłÆ╬▒)^w)┬Ę(RŌłÆQ)`. Audit logging is debounced ŌĆö same-action skips coalesce up to 10 cycles. All errors swallowed; quorum, confidence gate, daily loss budget, % drawdown breaker, kill switch are completely untouched. Endpoint: `GET /api/rl/execution`.
  - `services/knowledgeGraphService.js` ŌĆö long-term per-symbol company knowledge graph (Postgres `company_knowledge`). Stores company metadata, sector + peers, rolling earnings track (Ōēż6), valuation, macro context, and a curated major-event timeline (Ōēż6) extracted best-effort by Grok-mini. Daily refresh at 08:00 UTC + 30s post-boot warm-up; event-triggered staleness when sentiment |score|Ōēź0.5. Two-phase refresh persists deterministic core/summary BEFORE optional LLM enrichment, so the prompt block is never gated on LLM latency. Hard 1400-char summary cap. Injected as `knowledgeContext` in the LLM ensemble ŌĆö informational only, never affects quorum/gate/sizing. Endpoints: `GET /api/knowledge`, `GET /api/knowledge/:symbol`.
  - `services/backtestService.js` ŌĆö daily-bars rules-based backtest engine (RSI + trend + MACD entry, configurable stop/target/trailing). Models slippage (bps) and per-trade commission. Returns equity curve, trade log, Sharpe, max drawdown, win rate. Persists to `backtest_runs` table. Watchlist-only, in-flight lock, operator-gated.

## Strategy Configs (in `backend/strategies.js`)
- **Day** (US/USD via Alpaca): 1Min bars, 60s cadence, 0.5%/1% stop/target, $50ŌĆō$100 risk/trade, max 4 holdings, 3% position cap, auto-flatten 5m before close. No trailing stop (intraday only).
- **Swing** (US/USD via Alpaca): 15Min bars, 300s cadence, 2%/5% stop/target, $75ŌĆō$200 risk/trade, max 3 holdings, 5% position cap, can hold overnight. **Trailing stop**: arms once +2% above entry, then ratchets stop to `peak ├Ś (1 ŌłÆ 2.5%)` ŌĆö never moves down.
- **ASX Swing** (ASX/AUD via IBKR): 15Min bars, 600s cadence, 2.5%/6% stop/target, $75ŌĆō$200 risk/trade, max 3 holdings, 5% position cap, holds overnight, gated by Sydney market hours. Same trailing-stop rules as US swing.

## Multi-Market & FX
- One unified USD-equivalent virtual portfolio: ASX cash debits/credits = `native_price ├Ś qty ├Ś fx_at_trade`. Holdings store NATIVE qty + avg_cost + currency + fx_rate_at_entry; equity is summed in USD using a USD-equivalent price lookup so the **single $100/day USD loss budget and 5% drawdown circuit breaker apply across BOTH markets**.
- Risk sizing in USD-equivalent: `latest_native ├Ś fxToUsd` is fed to `evaluateBuy`; returned stop/take are USD-equivalent and converted back to native for both the broker order and `holdings` storage.
- Per-strategy market gating in `runCycle`: cycle proceeds when ANY market is open (so equity stays mark-to-market for the breaker), but each strategy only places new orders when its own market is open.
- Operator emergency controls fan out across brokers: kill-switch, cancel-all-orders, and flatten-all all hit BOTH Alpaca and IBKR via `brokerRouter.*AllBrokers` helpers (allSettled ŌĆö one broker failing doesn't skip the other).
- US-only enrichment services (premarket briefing, intraday tactical, 20-yr historical intelligence, options activity) are skipped for ASX symbols ŌĆö analyzer falls back to bar-derived signals (the safe degradation for a new market).

## Safety Features
- Server-side 85% confidence gate (cannot be bypassed by frontend).
- Circuit breaker auto-trips at 5% daily drawdown; requires manual reset.
- Emergency pause button halts all trading immediately.
- Ōēź3/4 LLMs must respond or system forces HOLD.
- Stop-loss skipped on stale price data (no fallback to avg cost).
- All decisions, trades, rejections, errors logged to `audit_log`.
- Allowlisted SQL column writes prevent injection via `updatePortfolio`.
- Atomic cash updates via `UPDATE ... SET cash = cash + delta` (no read-modify-write race).
- **Kill switch (Phase 4)**: `POST /api/agent/kill-switch` (operator-token + body `{confirm:"KILL"}`) trips a sticky in-memory flag set BEFORE any await, then cancels open orders, drains in-flight cycle (30s timeout), flattens all positions. Once tripped, `executeOrder()`, `runCycle()`, auto-hedge, AND `alpacaService.placeOrder()` itself all refuse new orders; `/api/agent/start` and `/run-now` return 409 "kill switch latched ŌĆö restart required". Only a process restart clears it.
- **Cancel all open orders (Phase 4)**: `POST /api/agent/cancel-orders` audit-logged broker order cancellation, separate from flatten.
- **Audit hash-chain (Phase 4)**: `audit_log` has `prev_hash` + `row_hash` (SHA-256 of `prev_hash || event_type || symbol || decision || JSON(payload) || created_at`), written inside a transaction with `LOCK TABLE audit_log IN EXCLUSIVE MODE`. `GET /api/audit/verify` walks the chain and reports the first break (or "ok"). Tamper-evident ŌĆö any row mutation invalidates every subsequent `row_hash`.
- **Compliance reports (Phase 4)**: `GET /api/audit/report?date=YYYY-MM-DD&format=json|csv` (operator-token gated). JSON includes daily P&L, win rate, model attribution, risk events (kill switch / circuit breaker / hedge / mode changes / drawdown pause), blocked-trade reasons (quorum/gate/risk), per-trade SIGNAL attachment via two-pass strategy-aware match (strict strategy === trade.strategy preferred, falls back to any-strategy within 5-min window), hash-chain verification result. CSV is a flat regulator-relevant trade sheet.
- **Asset-class scaffolding (Phase 4)**: `trades`, `holdings`, `audit_log` have `asset_class` column (default `'equity'`). Equities-only execution today; options/futures execution intentionally deferred (would require LLM prompt restructure, contracts├Śmultiplier sizing, different stop-loss math, and Alpaca's separate options contract symbol schema ŌĆö futures not supported by Alpaca at all).
